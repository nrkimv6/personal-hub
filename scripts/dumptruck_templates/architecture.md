# Architecture 초안 요청 (Gemini 3.1 Pro oneshot)

당신은 대규모 코드베이스 아키텍처 설계자입니다. 아래 첨부된 디렉토리 트리와 파일 전문을 바탕으로, **현 구조의 아키텍처 초안 문서**를 작성하세요.

## 작성 지침

- 코드만 보고 추측 가능한 사실만 적습니다. 추측이 필요하면 "추정:" 접두어를 붙입니다.
- 한국어 마크다운, 섹션 헤더는 `##`/`###`을 사용합니다.
- 코드 라인 인용은 `path:line` 형식.

## 답할 질문 (필수)

1. **레이어 구조**: 진입점(routes/CLI/worker) → 서비스 → 데이터 계층의 의존 방향을 ASCII 다이어그램으로.
2. **핵심 도메인 모델**: 가장 자주 참조되는 데이터 모델 5개와 그 책임.
3. **확장 포인트(Seam)**: 새 기능 추가 시 가장 흔히 건드려야 하는 파일/함수 3~5개.
4. **위험 신호**: 순환 의존, God 객체, 깨진 추상화가 의심되는 지점 (각각 근거 코드 위치 포함).
5. **권장 다음 리팩터링 1건**: 가장 ROI가 높은 단일 리팩터링과 그 이유.

## 출력 형식

```
# {프로젝트명} 아키텍처 초안

> 생성일: YYYY-MM-DD
> 모델: gemini-3.1-pro (oneshot)
> 입력 파일 수: N

## 1. 레이어 구조
...
```
