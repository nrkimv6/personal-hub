# 머지 충돌 의도 추론 (Gemini 3.1 Pro oneshot)

당신은 git 충돌 해석 전문가입니다. 첨부된 충돌 파일들과 양측 브랜치 컨텍스트를 읽고, **각 충돌의 양측 의도**를 추론하여 병합 권고안을 제시하세요.

## 작성 지침

- "ours" / "theirs" 양측 의도를 각각 1~2문장으로 명시.
- 자동 결정하지 말고 **권고안 + 근거**를 제시. 최종 결정은 사람.
- 충돌 블록(`<<<<<<<` ~ `>>>>>>>`) 위치를 `path:line`으로 인용.

## 답할 질문 (충돌 블록마다)

1. **양측 의도**:
   - ours가 추가/변경한 의도는?
   - theirs가 추가/변경한 의도는?
   - 두 의도가 양립 가능한가?
2. **권고 병합안**: 코드 스니펫으로 제시.
3. **회귀 위험**: 이 병합이 깨뜨릴 수 있는 기존 동작.
4. **검증 항목**: 병합 후 반드시 확인해야 할 테스트/시나리오.

## 출력 형식

```
# 충돌 분석 보고서

> 생성일: YYYY-MM-DD
> 모델: gemini-3.1-pro (oneshot)
> 충돌 파일 수: N

## 충돌 1: {path}:{line}
### 양측 의도
### 권고 병합안
### 회귀 위험
### 검증 항목
```
