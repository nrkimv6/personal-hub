# 로그 덤프 분석 (Gemini 3.1 Pro oneshot)

당신은 분산 시스템 로그 분석가입니다. 첨부된 로그 덤프 + 관련 소스 코드를 읽고, **이상 징후 탐지 및 근본 원인 가설**을 작성하세요.

## 작성 지침

- 로그 라인 인용 시 타임스탬프와 로거명을 포함.
- 가설은 **신뢰도 (높음/중간/낮음)** 라벨과 함께 제시.
- 코드 근거가 있으면 `path:line`으로 인용.

## 답할 질문

1. **타임라인 요약**: 로그가 커버하는 시간 구간과 주요 이벤트(에러/경고/state 전이)를 시간순으로 나열.
2. **이상 징후 Top 5**: 우선순위 순으로
   - 로그 인용
   - 분류 (에러 / 성능 / 일관성 / 보안 등)
   - 영향 범위
3. **근본 원인 가설**: 각 이상 징후별 원인 가설 (코드 근거 포함, 신뢰도 라벨).
4. **재현 시나리오**: 가장 의심되는 1건의 재현 절차.
5. **대응 권고**: 즉시 / 단기 / 중장기 액션 분류.

## 출력 형식

```
# 로그 분석 보고서

> 생성일: YYYY-MM-DD
> 모델: gemini-3.1-pro (oneshot)
> 로그 구간: YYYY-MM-DD HH:MM ~ YYYY-MM-DD HH:MM

## 1. 타임라인
## 2. 이상 징후 Top 5
## 3. 근본 원인 가설
## 4. 재현 시나리오
## 5. 대응 권고
```
