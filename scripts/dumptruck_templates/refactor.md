# 대규모 리팩터 가이드 요청 (Gemini 3.1 Pro oneshot)

당신은 시니어 리팩터링 컨설턴트입니다. 첨부된 코드를 읽고 **단계별 리팩터 가이드**를 작성하세요.

## 작성 지침

- 코드 근거 없는 일반론 금지. 모든 권고는 `path:line` 인용을 동반합니다.
- 단일 거대 PR이 아닌 **순차 적용 가능한 작은 단계**로 분해.
- 각 단계는 독립적으로 머지/롤백 가능해야 합니다.

## 답할 질문

1. **현 상태 진단**: 가장 큰 리팩터링 부채 3가지 (각각 영향 범위 + 근거 파일).
2. **목표 상태**: 리팩터링 완료 후의 구조를 ASCII 다이어그램으로.
3. **단계별 마이그레이션 계획**: 5~10개 단계, 각 단계는
   - 변경 대상 파일 목록
   - 변경 내용 요약 (1~2문장)
   - 검증 방법 (어떤 테스트가 통과해야 하는지)
   - 롤백 시나리오
4. **위험 요소**: 각 단계별 잠재적 회귀 지점.
5. **선행 준비**: 본 리팩터 착수 전 추가해야 할 테스트 커버리지.

## 출력 형식

```
# {대상 모듈} 리팩터 가이드

> 생성일: YYYY-MM-DD
> 모델: gemini-3.1-pro (oneshot)

## 1. 진단
## 2. 목표 상태
## 3. 단계별 계획
### Step 1: ...
### Step 2: ...
...
```
