# 웹 페이지 모니터링 시스템 PRD

## 1. 개요
이 시스템은 여러 웹 페이지의 변경사항을 모니터링하고 변경이 감지되면 알림을 보내는 자동화 시스템입니다. 페이지 내용의 변경을 감지하고, 변경사항이 있을 경우 텔레그램과 데스크톱 알림을 통해 사용자에게 알립니다.

## 2. 주요 기능

### 2.1 페이지 모니터링
- 여러 웹 페이지를 동시에 모니터링
- 각 페이지의 내용 변경 감지 (페이지 해시값 비교)
- 변경사항 발생 시 텔레그램 알림 발송
- 데스크톱 알림 지원
- 페이지 로딩 타임아웃 설정 (30초)
- 네트워크 오류 시 자동 재시도
- 에러 페이지 감지 및 처리:
  - 리다이렉트된 에러 페이지 감지
  - 에러 페이지 발생 시 로깅 및 알림
  - 에러 페이지에서 정상 페이지로 복구 시 자동 재시도
  - 에러 타입별 분류 및 로깅
- 시간 및 매수 정보 파싱:
  - 버튼 텍스트에서 시간 정보 추출 (오전/오후 HH:MM)
  - 버튼 텍스트에서 매수 정보 추출 (숫자+매)
  - 텔레그램 알림은 시간 변경만 포함
  - 데스크톱 알림은 시간과 매수 변경 모두 포함

### 2.2 스케줄링 시스템
- URL의 날짜 파라미터를 기반으로 모니터링 간격 자동 계산
- 우선순위 기반 작업 스케줄링 (힙 큐 사용)
- 날짜별 모니터링 간격:
  - 과거 날짜: 2-7초
  - 1-7일 후: 20-50초
  - 7일 이상: 200-300초
  - 내일: 3-10초
- 스케줄러 등록 프로세스:
  1. URL에서 날짜 파라미터 추출
  2. 날짜 기반으로 모니터링 간격 계산
  3. 계산된 간격으로 다음 실행 시간 설정
  4. 우선순위 큐에 작업 등록
  5. 설정된 시간에 작업 실행
- 프로그램 시작 시 알림:
  - 데스크톱 알림으로 프로그램 시작 알림
  - 모니터링 대상 URL 수 표시
  - 워커 수 및 설정 정보 표시
  - 시작 시간 표시

### 2.3 중복 메시지 필터링
- 최근 10개의 메시지 기록 유지
- 메시지 중복 판단 기준:
  - LABEL (태그) 동일
  - TIMES (시간대) 동일
- 중복 메시지 처리:
  - 텔레그램 알림 발송하지 않음
  - 데스크톱 알림만 발송
- 메시지 기록 관리:
  - 최대 10개 메시지 유지
  - FIFO(First In First Out) 방식으로 오래된 메시지 제거

## 3. 구현 방식

### 3.1 browser7.py
- 멀티스레딩 방식으로 구현
- 주요 컴포넌트:
  - 워커 스레드: 페이지 모니터링 담당
    - 브라우저 인스턴스 생성 및 관리
    - 페이지 접근 및 내용 추출
    - 변경 감지 및 알림 발송
  - 스케줄러 스레드: 작업 스케줄링 담당
    - 우선순위 큐 관리
    - 작업 실행 시간 계산
    - 작업 큐에 작업 추가
  - 이벤트 루프 스레드: 비동기 작업 처리
    - 텔레그램 알림 비동기 발송
    - 데스크톱 알림 발송
- 브라우저 프로필 관리:
  - 사용자 프로필 디렉토리 사용 (로그인 상태 유지)
  - 워커별 독립적인 프로필 디렉토리 사용
  - 브라우저 세션 유지 (창 상태, 로그인 상태 등)
  - 브라우저 인스턴스 재사용으로 성능 최적화
  - 브라우저 컨텍스트 지속성 유지
  - 워커별 Playwright 인스턴스 관리
  - 프로필 복사 및 초기화 메커니즘
  - 브라우저 창 상태 보존
  - 탭 관리 최적화:
    - 워커별 고정된 탭 풀 유지
    - 탭 재사용으로 메모리 사용량 최적화
    - 탭 수 제한 (워커당 최대 5개)
    - 오래된 탭 자동 정리
    - 탭 상태 모니터링 및 복구
- Chrome Dev 브라우저 사용
- 브라우저 자동화 감지 방지 설정

### 3.2 run_playwright.py (Playwright 기반)
- 비동기(asyncio) 방식으로 구현
- 주요 기능:
  - 브라우저 상태 유지 (로그인 세션 등)
  - 페이지별 독립적인 모니터링 태스크
  - 자동 재시도 메커니즘
- 브라우저 설정:
  - 자동화 감지 방지
  - 샌드박스 비활성화
  - GPU 가속 비활성화
  - 뷰포트 크기 설정
- 태스크 관리:
  - 각 URL별 독립적인 태스크 생성
  - 태스크별 상태 관리
  - 태스크 오류 시 자동 재시도

## 4. 공통 기능

### 4.1 URL 처리
- URL에서 날짜 파라미터 추출
- 날짜 형식 지원:
  - ISO 형식 (YYYY-MM-DDTHH:MM:SS+ZZZZ)
  - 기본 날짜 형식 (YYYY-MM-DD)
- URL 파싱 오류 처리:
  - 파싱 실패 시 기본 간격 사용
  - 오류 로깅 및 알림

### 4.2 알림 시스템
- 텔레그램 알림
  - 변경사항 감지 시 자동 발송
  - 에러 발생 시 알림
  - 중복 메시지 필터링 적용
  - 시간 변경 정보만 포함 (매수 정보 제외)
- 데스크톱 알림
  - 시스템 레벨 알림 지원
  - 모든 변경사항에 대해 발송 (중복 필터링 없음)
  - 시간과 매수 변경 정보 모두 포함
- 알림 메시지 형식:
  - 태그 정보
  - 변경 내용
  - 시간 정보
  - 변경 감지 시간

### 4.3 유효성 검사
- 콘텐츠 유효성 검사 레벨 설정 가능
  - 레벨 0: 검사 없음
  - 레벨 1: 기본 검사
  - 레벨 2: 상세 검사
- 비활성 페이지 메시지 감지
- 중복 알림 방지
- 콘텐츠 비교 로직:
  - 해시값 비교
  - 콘텐츠 직접 비교 (설정에 따라)

## 5. 에러 처리
- 브라우저 생성 실패 시 재시도
  - 최대 3회 재시도
  - 재시도 간격 5초
  - 브라우저 컨텍스트 오류 시 자동 재생성
  - Playwright 인스턴스 오류 시 자동 재시작
- 네트워크 오류 시 자동 재시도
  - 타임아웃 설정 (30초)
  - 재시도 간격 10초
- 날짜 파싱 오류 시 기본 간격 사용
  - 기본 간격: 2-7초
- 예외 상황 로깅 및 알림
  - 오류 메시지 상세 기록
  - 심각한 오류 시 텔레그램 알림
  - 브라우저 컨텍스트 오류 로깅
  - 프로필 관련 오류 로깅
- 에러 페이지 처리
  - 에러 페이지 URL 패턴 감지
  - 에러 페이지 발생 시 로깅
  - 에러 페이지 발생 시 알림 발송
  - 에러 페이지에서 정상 페이지로 복구 시 자동 재시도
  - 에러 페이지 발생 빈도 모니터링
  - 에러 타입별 분류 및 로깅

## 6. 설정
- 사용자 프로필 디렉토리 설정
  - 기본 경로: C:\Users\[사용자]\AppData\Local\Google\Chrome\User Data
- Chrome 브라우저 경로 설정
  - Chrome Dev 경로: C:\Program Files\Google\Chrome Dev\Application\chrome.exe
- ChromeDriver 경로 설정
  - 버전별 경로 설정
  - 자동 업데이트 지원
- 알림 설정 (텔레그램, 데스크톱)
  - 텔레그램 봇 토큰 설정
  - 채팅 ID 설정
  - 알림 간격 설정
- 중복 메시지 필터링 설정
  - 최대 저장 메시지 수 (기본값: 10)
  - 중복 판단 기준 : label, 시각이 일치할 경우
