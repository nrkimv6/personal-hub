# Dev Runner 페이지 리디자인 — Design Prompt

## 페이지 탭

| 탭 | 설명 |
|----|------|
| **Dev Runner** | 이 문서의 대상 |
| **Sleep Now** | 예약 종료 |
| **Git 관리** | git 레포 상태/조작 |
| **Plan 관리** | plan 목록 + 상세 뷰 |
* Dev Runner 이외의 다른 기능은 고려하지 않는다

---

## Dev Runner 요구사항

### 원칙
- 모바일 우선 심플 디자인, 스크롤 최소화
- 각 버튼은 안내 글자를 노출하지 않거나 최소한으로+툴팁
- 이모지 금지, 색상이 들어간 특수문자나 단색 아이콘 라이브러리 사용가능
- 실행 중일 때 로그에 최대 공간 할당, 불필요한 영역은 자동 접힘
- 로그 뷰 최소 높이 5줄 보장, 데스크톱은 특히 세로로 길게


### 실행 로그
- 여러 runner가 동시에 실행될 수 있다
- 특정 runner의 로그를 선택해서 볼 수 있다 (방식은 탭 또는 필터 등 자유)
- 실시간 스트리밍 로그
- 각 로그 줄에 timestamp, tag(AI/TOOL/DONE/ERROR/SYSTEM/PHASE/CYCLE 등), message가 보인다. tag별 시각적 구분
- 사이클/페이즈 전환 시 구분 표시가 보인다
- 로그는 기본적으로 실시간(하단고정) 이지만 버튼으로 해제할 수 있다. 해제시 마지막으로 스크롤 한 뒤 쌓인 로그 줄 수가 bullet으로 표시됨
- 노이즈 라인은 접혀서 `...` 로 보이고 펼칠 수 있다
- runner 완료 시 완료 표시가 보인다
- merge queue 로그도 단독으로 볼 수 있다

### Plan 목록
- 각 plan(일반적으로 md형식의 문서)에 파일명, 상태(초안/검토완료/구현중/구현완료/보류 등), 진행률(done/total)이 보인다 - 접혀도 간략하게 표시됨
- 실행상태 : 실행 중인 plan은 실행 중 표시가 된다
- plan을 보류/보류 해제할 수 있다
- plan을 무시할 수 있다. 무시된 plan 목록을 볼 수 있고 무시를 해제할 수 있다
- plan 저장 경로(파일 또는 폴더)를 추가/제거할 수 있다
- 완료 가능한 plan을 개별 또는 일괄로 완료 처리할 수 있다 (아카이브 이동, TODO→DONE, 커밋)
- 코드베이스 검증 기반 일괄 완료 처리를 할 수 있다

### 실행 설정
- plan을 선택 시 전환/팝업 등으로 노출된다.
- 상단에 요약(1-3문장)이 노출된다,
- phase별 모델을 선택할 수 있다: plan / impl / done 각각 provider:model
  - 각 드롭다운에 모든 provider의 모델이 표시된다 (예: "Claude Opus", "Gemini 3 Pro")
- max cycles를 입력할 수 있다
- until(종료 시각)을 입력할 수 있다
- dry run을 토글할 수 있다
- 전체 실행 모드를 선택할 수 있다 (plan 지정 없이 미완료 plan 전체 자동 실행)
- 병렬 실행을 토글할 수 있다
- 대상 프로젝트를 지정할 수 있다 (쉼표 구분, 비우면 전체)
- [실행] 할 수 있다

### 실행 상태
- 실행중 runner 수, 실행시간 정보가 보인다
- 각 runner에 plan 파일명, engine, runner ID, worktree 브랜치(있으면)가 보인다
- SSE 연결 상태, Redis 연결 상태가 색상 아이콘으로 보인다
- 비정상 종료(crashed) 상태가 보인다
- 머지 충돌 시 충돌 표시가 보이고, 머지 재시도/worktree 정리를 할 수 있다
- 전체 중지할 수 있다 (runner 존재 및 실행 시)
- 개별 runner를 중지/강제종료(kill)할 수 있다
- 강제 중지할 수 있다 (상태가 실행 중으로 잘못 남아있을 때)
- 연결 끊김 시 수동 재연결할 수 있다
- 동기화, 초기화, 전체 리셋을 할 수 있다


### Tasks
- 실행 설정 -> 실행 시  전환/팝업 등으로 노출된다.
- 실행 중인 plan의 요약내용(1-3문장)과  체크박스 항목(TODO)이 보인다
- 실시간 갱신된다

### 현재 추적 태스크
- 실행 중일 때, AI가 작업 중인 태스크 텍스트가 강조된다
- confidence 수준(HIGH/MEDIUM/LOW)이 보인다
- 추적 정보가 오래되면 stale로 표시된다

### Merge Queue
- 태스크 마무리 단계에서 전환/팝업 등으로 노출된다.
- 머지 대기 항목 목록이 보인다. 각 항목에 plan 파일명, 프로젝트명, 브랜치명, 상태(대기/머지 중/테스트 중/수정 중/완료/실패), 타임스탬프가 표시된다
- 실패한 항목은 재시도할 수 있다
- 실패한 항목은 되돌리기(revert)할 수 있다.  머지큐에 사라지지 않고 실패 상태로 남음
- 항목을 선택하면 머지 로그가 실시간으로 보인다 (tag별 시각 구분: COMMIT/MERGE/TEST/ERROR/DONE)
